Konsep yang Digunakan

Material Design 3 (M3)

Material Design 3 adalah sistem desain dari Google dengan tiga prinsip utama:

  • Material as Metaphor - UI dianalogikan seperti benda nyata. Card dengan elevation menciptakan bayangan yang memberi kesan kartu melayang. ExtendedFloatingActionButton terlihat mengambang di atas konten.
  • Bold, Graphic, Intentional - Warna kontras, tipografi jelas, hierarki visual yang menekankan konten penting. Harga produk ditampilkan dengan FontWeight.Bold berwarna primary.
  • Motion Provides Meaning - Animasi membantu user memahami perubahan UI. Transisi antar layar menggunakan animasi default Navigation Compose. Ripple effect pada setiap komponen interaktif.

M3 menggunakan sistem warna pasangan: primary + onPrimary (warna teks di atas primary), primaryContainer + onPrimaryContainer (untuk area berwarna lembut seperti TopAppBar).

Jetpack Compose

Framework UI deklaratif Android modern. Alih-alih memanipulasi View secara imperatif, kita mendeskripsikan tampilan sebagai fungsi dari state. Ketika state berubah, Compose otomatis menggambar ulang (recompose) hanya bagian yang terpengaruh.

ViewModel + StateFlow

ViewModel menyimpan dan mengelola UI state sehingga bertahan saat Activity diputar ulang (configuration change). StateFlow adalah aliran data reaktif yang selalu memiliki nilai terbaru — UI cukup mengamati (observe) flow ini dan otomatis diperbarui.

Navigation Compose

Library navigasi untuk Jetpack Compose. Setiap layar didefinisikan sebagai rute (string), dan NavController mengelola back stack perpindahan antar layar.

data/Product.kt

data class Product(
    val id: Int,
    val title: String,
    val price: Int,
    val description: String,
    val category: String,
    val seller: String,
    val condition: String,
    val emoji: String
)

data class di Kotlin secara otomatis menghasilkan equals(), hashCode(), toString(), dan copy(). Fungsi copy() berguna untuk membuat salinan objek dengan beberapa field berbeda tanpa mengubah yang lain.

sampleProducts adalah List<Product> top-level berisi 8 produk contoh. Ini berfungsi sebagai "database" statis karena proyek ini belum menggunakan Room.

viewmodel/MarketplaceViewModel.kt

data class MarketplaceUiState(
    val products: List<Product> = sampleProducts,
    val searchQuery: String = "",
    val selectedCategory: String = "Semua",
    val filteredProducts: List<Product> = sampleProducts
)

class MarketplaceViewModel : ViewModel() {
    private val _uiState = MutableStateFlow(MarketplaceUiState())
    val uiState: StateFlow<MarketplaceUiState> = _uiState.asStateFlow()

    private fun applyFilter() {
        val state = _uiState.value
        val filtered = sampleProducts.filter { product ->
            val matchesSearch = product.title.contains(state.searchQuery, ignoreCase = true) ||
                    product.description.contains(state.searchQuery, ignoreCase = true)
            val matchesCategory = state.selectedCategory == "Semua" || product.category == state.selectedCategory
            matchesSearch && matchesCategory
        }
        _uiState.value = state.copy(filteredProducts = filtered)
    }
}

MutableStateFlow dibuat private agar hanya ViewModel yang bisa mengubah state (enkapsulasi). UI hanya menerima StateFlow read-only via asStateFlow().

Fungsi applyFilter() menerapkan filter dengan logika AND: produk harus memenuhi kondisi pencarian (di title atau description) DAN kondisi kategori. ignoreCase = true membuat pencarian tidak case-sensitive. state.copy(filteredProducts = filtered) membuat salinan state baru hanya dengan mengganti field filteredProducts.

ui/theme/Theme.kt

private val LightColors = lightColorScheme(
    primary = Color(0xFF6750A4),
    onPrimary = Color.White,
    primaryContainer = Color(0xFFEADDFF),
    secondary = Color(0xFF625B71),
    tertiary = Color(0xFF7D5260),
    background = Color(0xFFFFFBFE),
    surface = Color(0xFFFFFBFE),
)

@Composable
fun MarketplaceSiswaTheme(content: @Composable () -> Unit) {
    MaterialTheme(colorScheme = LightColors, content = content)
}

Setiap warna M3 memiliki pasangan: primary adalah warna aksi utama (tombol, ikon aktif), onPrimary adalah warna teks/ikon yang diletakkan di atasnya (selalu kontras). primaryContainer digunakan untuk area berwarna lembut seperti TopAppBar.

MarketplaceSiswaTheme adalah wrapper composable yang menerapkan skema warna ke seluruh hierarki komponen di dalamnya melalui MaterialTheme.

ui/screens/HomeScreen.kt

Scaffold(
    topBar = { TopAppBar(...) },
    floatingActionButton = { ExtendedFloatingActionButton(...) }
) { padding ->
    Column {
        OutlinedTextField(...)       // Search bar
        LazyRow { FilterChip(...) } // Chip kategori
        LazyVerticalGrid(columns = GridCells.Fixed(2)) {
            items(uiState.filteredProducts) { ProductCard(...) }
        }
    }
}

Scaffold adalah layout dasar Material 3 yang menangani penempatan TopAppBar, FAB, dan padding konten secara otomatis. Parameter { padding -> } berisi padding yang harus diaplikasikan ke konten agar tidak tertimpa TopAppBar.

LazyRow dan LazyVerticalGrid hanya merender item yang terlihat di layar (lazy loading), sehingga efisien untuk daftar panjang.

FilterChip dengan parameter selected = uiState.selectedCategory == category otomatis mengubah tampilannya (filled vs outlined) berdasarkan state tanpa kode tambahan.

OutlinedTextField bersifat stateless — state (searchQuery) disimpan di ViewModel, bukan di dalam composable. viewModel::onSearchQueryChange adalah method reference sebagai callback onValueChange.

ui/screens/ProductDetailScreen.kt

var isFavorite by remember { mutableStateOf(false) }

Icon(
    if (isFavorite) Icons.Default.Favorite else Icons.Default.FavoriteBorder,
    tint = if (isFavorite) MaterialTheme.colorScheme.error else MaterialTheme.colorScheme.onSurface
)

remember { mutableStateOf(false) } menyimpan state lokal di dalam composable. Berbeda dengan state di ViewModel, state ini hanya hidup selama composable ada di komposisi. Cocok untuk state yang tidak perlu di-share antar layar.

by (property delegation) memungkinkan akses langsung sebagai isFavorite alih-alih isFavorite.value.

Icons.AutoMirrored.Filled.ArrowBack — versi "auto-mirrored" secara otomatis membalik ikon untuk layout RTL (Right-to-Left, seperti bahasa Arab).

@Composable
private fun InfoRow(label: String, value: String) {
    Row(horizontalArrangement = Arrangement.SpaceBetween) {
        Text(label, color = MaterialTheme.colorScheme.onSurfaceVariant)
        Text(value, fontWeight = FontWeight.Medium)
    }
}

Composable private yang digunakan berulang untuk menampilkan pasangan label-nilai. Ini contoh reusability di Compose — logic yang berulang diekstrak menjadi fungsi tersendiri.

MainActivity.kt

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContent {
            MarketplaceSiswaTheme {
                val navController = rememberNavController()
                val viewModel: MarketplaceViewModel = viewModel()

                NavHost(navController = navController, startDestination = "home") {
                    composable("home") {
                        HomeScreen(viewModel = viewModel, onProductClick = { product ->
                            navController.navigate("detail/${product.id}")
                        })
                    }
                    composable("detail/{productId}") { backStack ->
                        val productId = backStack.arguments?.getString("productId")?.toIntOrNull()
                        val product = sampleProducts.find { it.id == productId }
                        if (product != null) {
                            ProductDetailScreen(product = product, onBack = { navController.popBackStack() })
                        }
                    }
                }
            }
        }
    }
}

enableEdgeToEdge() membuat konten meluas ke area status bar dan navigation bar, memberikan tampilan modern full-screen.

val viewModel: MarketplaceViewModel = viewModel() — ViewModel dibuat di level MainActivity agar satu instance digunakan oleh semua layar. ViewModel bertahan melewati configuration change (rotasi layar).

composable("detail/{productId}") mendefinisikan rute dengan parameter dinamis. Parameter diakses lewat backStack.arguments?.getString("productId"). ?.toIntOrNull() safely mengonversi string ke Int, mengembalikan null jika gagal.

Project 5 - News App (REST API & Retrofit)

Referensi: REST API – kuliahppb.blogspot.com

Aplikasi berita modern yang mengambil data dari REST API (NewsAPI) menggunakan Retrofit untuk komunikasi HTTP dan menampilkannya dengan Jetpack Compose. Arsitektur yang digunakan adalah MVVM dengan StateFlow dan Kotlin Coroutines sebagai lapisan reaktif. Pengguna dapat melihat berita utama di Home, membaca detail artikel, dan mencari berita berdasarkan kata kunci.

Konsep yang Digunakan

REST API & HTTP GET

REST API adalah standar komunikasi antara aplikasi Android (client) dan server menggunakan protokol HTTP. Proyek ini menggunakan dua endpoint dari NewsAPI:

  • GET https://newsapi.org/v2/top-headlines?country=us&apiKey=... — mengambil berita utama untuk layar Home.
  • GET https://newsapi.org/v2/everything?q=keyword&apiKey=... — mencari berita berdasarkan kata kunci di layar Search.

Server merespons dengan JSON yang di-parse otomatis oleh Gson menjadi data class Kotlin (NewsResponse, Article).

Retrofit + Gson Converter

Retrofit adalah library HTTP client untuk Android yang mengubah antarmuka Kotlin (ApiService) menjadi HTTP request secara otomatis. GsonConverterFactory bertugas mengkonversi JSON response menjadi objek Kotlin. OkHttp HttpLoggingInterceptor digunakan untuk mencetak log HTTP request/response saat debugging.

Arsitektur MVVM

Alur data: UI (Composable) → ViewModel → Repository → ApiService → REST API. Setiap lapisan hanya bergantung pada lapisan di bawahnya — UI tidak pernah menyentuh Retrofit secara langsung. ViewModel menyimpan state UI, Repository bertugas mengambil data dan menangani error, ApiService mendefinisikan endpoint.

StateFlow + Coroutines

StateFlow<NewsUiState> menyimpan state UI yang dapat diobservasi. Setiap kali data berubah, UI otomatis merender ulang. Pemanggilan API dijalankan di viewModelScope dengan Dispatchers.IO agar tidak memblokir Main thread.

Sealed Interface sebagai UI State (Loading / Success / Error)

NewsUiState dimodelkan sebagai sealed interface dengan tiga kemungkinan: Loading, Success(articles), dan Error(message). Pola ini memaksa UI untuk menangani semua kemungkinan state secara eksplisit di blok when, sekaligus menghilangkan kebutuhan field isLoading: Boolean dan errorMessage: String? yang terpisah.

Coil — Async Image Loading

Coil (coil-compose) digunakan untuk memuat gambar dari URL (urlToImage) secara asinkron ke dalam AsyncImage. Coil menangani caching, placeholder, dan error secara otomatis sehingga kode UI tetap ringkas.

Navigasi dengan NavHost

Tiga rute didefinisikan: home, search, dan detail. Satu NewsViewModel dibagikan ke semua screen sehingga artikel yang dipilih dapat dibaca di DetailScreen tanpa harus dikirim sebagai argumen navigasi.

data/api/ApiService.kt

interface ApiService {
    @GET("v2/top-headlines")
    suspend fun getTopHeadlines(
        @Query("country") country: String = "us",
        @Query("category") category: String? = null,
        @Query("apiKey") apiKey: String
    ): NewsResponse

    @GET("v2/everything")
    suspend fun searchNews(
        @Query("q") query: String,
        @Query("sortBy") sortBy: String = "publishedAt",
        @Query("apiKey") apiKey: String
    ): NewsResponse
}

Interface ini adalah "kontrak" API yang dibaca Retrofit untuk membuat implementasi HTTP secara otomatis. Anotasi @GET menentukan path endpoint, @Query menambahkan query parameter ke URL. Fungsi bersifat suspend sehingga dapat dipanggil dari dalam Coroutine tanpa memblokir thread.

data/api/RetrofitClient.kt

object RetrofitClient {
    // TODO: Ganti dengan API key dari https://newsapi.org/register
    const val API_KEY = "YOUR_API_KEY_HERE"
    private const val BASE_URL = "https://newsapi.org/"

    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor(HttpLoggingInterceptor().apply {
            level = HttpLoggingInterceptor.Level.BODY
        })
        .build()

    val apiService: ApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(GsonConverterFactory.create())
            .build()
            .create(ApiService::class.java)
    }
}

object Kotlin memastikan hanya ada satu instance Retrofit di seluruh aplikasi (singleton). by lazy membuat apiService baru dibuat saat pertama kali diakses, bukan saat aplikasi launch — menghemat resource startup. LoggingInterceptor dengan level BODY mencetak seluruh request dan response ke Logcat saat debugging.

Penting: Ganti nilai API_KEY dengan kunci dari newsapi.org/register sebelum menjalankan aplikasi.

data/repository/NewsRepository.kt

class NewsRepository {
    suspend fun getTopHeadlines(country: String = "us"): Result<List<Article>> {
        return try {
            val response = api.getTopHeadlines(country = country, apiKey = apiKey)
            if (response.status == "ok") {
                Result.success(response.articles)
            } else {
                Result.failure(Exception("API error: ${response.status}"))
            }
        } catch (e: Exception) {
            Result.failure(Exception("Failed to load news: ${e.message}"))
        }
    }
}

Repository adalah satu-satunya tempat yang menyentuh ApiService. Blok try/catch menangkap semua kemungkinan error: timeout jaringan, JSON parse error, HTTP 4xx/5xx, dll — semuanya dibungkus menjadi Result.failure() dengan pesan yang ramah. ViewModel tidak perlu tahu bagaimana data diambil, hanya menerima Result<List<Article>>.

viewmodel/NewsViewModel.kt

sealed interface NewsUiState {
    object Loading : NewsUiState
    data class Success(val articles: List<Article>) : NewsUiState
    data class Error(val message: String) : NewsUiState
}

class NewsViewModel : ViewModel() {
    private val _homeState = MutableStateFlow<NewsUiState>(NewsUiState.Loading)
    val homeState: StateFlow<NewsUiState> = _homeState.asStateFlow()

    fun loadTopHeadlines() {
        viewModelScope.launch(Dispatchers.IO) {
            _homeState.value = NewsUiState.Loading
            val result = repository.getTopHeadlines()
            _homeState.value = result.fold(
                onSuccess = { NewsUiState.Success(it) },
                onFailure = { NewsUiState.Error(it.message ?: "Unknown error") }
            )
        }
    }
}

sealed interface NewsUiState membuat tiga state yang saling eksklusif. Blok when(state) di UI harus menangani semua cabang — compiler mewajibkannya, sehingga tidak ada state yang terlewat. MutableStateFlow yang di-expose sebagai StateFlow (read-only) melindungi state dari modifikasi luar. viewModelScope.launch(Dispatchers.IO) menjalankan network call di thread IO pool, hasilnya dikembalikan ke Main thread melalui StateFlow.

selectArticle(article) menyimpan artikel yang dipilih ke _selectedArticle: MutableStateFlow<Article?> sehingga DetailScreen dapat membacanya tanpa perlu meneruskan objek Article sebagai argumen navigasi (yang memerlukan Parcelable).

ui/screens/HomeScreen.kt

when (val state = uiState) {
    is NewsUiState.Loading -> {
        CircularProgressIndicator()
        Text("Loading...")
    }
    is NewsUiState.Success -> {
        LazyColumn {
            item { FeaturedNewsCard(articles.first(), ...) }
            item { Text("Breaking News") }

    }
}

Layar utama menampilkan satu FeaturedNewsCard untuk artikel pertama, diikuti header "Breaking News" dan daftar artikel selanjutnya menggunakan NewsListItem. Tombol Refresh di TopAppBar memanggil viewModel.refresh() yang identik dengan loadTopHeadlines(). Saat API gagal, blok Error menampilkan pesan error dan tombol Retry.

util/DateUtil.kt

fun formatPublishedAt(publishedAt: String?): String {
    val date = SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ss'Z'", Locale.US).parse(publishedAt)
    val diffMin = (Date().time - date.time) / 60_000
    return when {
        diffMin < 60   -> "${diffMin}m ago"
        diffMin < 1440 -> "${diffMin / 60}h ago"
        else           -> "${diffMin / 1440}d ago"
    }
}

Mengkonversi timestamp ISO-8601 dari NewsAPI (misal "2026-06-15T09:00:00Z") menjadi label relatif seperti "3h ago" atau "2d ago". try/catch menangani format tidak valid — jika gagal di-parse, dikembalikan 10 karakter pertama string (tanggal kasar).

MainActivity.kt

val navController = rememberNavController()
val viewModel: NewsViewModel = viewModel()

NavHost(navController = navController, startDestination = "home") {
    composable("home") {
        HomeScreen(viewModel, onArticleClick = { navController.navigate("detail") }, ...)
    }
    composable("search") {
        SearchScreen(viewModel, onArticleClick = { navController.navigate("detail") }, ...)
    }
    composable("detail") {
        val article = viewModel.selectedArticle.value
        if (article != null) DetailScreen(article, onBack = { navController.popBackStack() })
    }
}

Satu instance NewsViewModel dibuat di MainActivity dan diteruskan ke semua screen — ini memungkinkan berbagi state (terutama selectedArticle) antar rute tanpa serialisasi Parcelable. navController.popBackStack() di DetailScreen dan SearchScreen mengembalikan user ke layar sebelumnya.
